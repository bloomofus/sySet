# 本地网盘搭建

## 安装Cloudreve

首先安装好Cloudreve 

可以使用一下的bat脚本隐身启动：

```bat
@echo off
powershell -WindowStyle Hidden -Command "Start-Process 'E:\WorkSpace\cloudreve_4.11.1_windows_amd64\cloudreve.exe'  -WindowStyle Hidden"

```

Cloudreve 的端口是5212，第一个注册的账号是管理员。

## IP固定

局域网最好设置固定IP地址，方法如下：

> 1. **打开网络设置**
>
>    - 按 `Win + R`，输入 `ncpa.cpl`，回车
>
> 2. **选择网络适配器**
>
>    - 右键点击正在使用的网络连接（如"以太网"或"WLAN"）
>    - 选择 **"属性"**
>
> 3. **配置 IPv4**
>
>    - 双击 **"Internet 协议版本 4 (TCP/IPv4)"**
>
> 4. **设置静态 IP**
>

设置固定IP地址之后，如果连接手机热点，可能出现网络问题，需要注意。

## 挂载为本地磁盘

### 开启WebDAV服务器

![image-20260127204216545](./%E6%9C%AC%E5%9C%B0%E7%BD%91%E7%9B%98%E6%90%AD%E5%BB%BA.assets/image-20260127204216545.png)

### 安装步骤

#### 1. 下载安装两个软件

| 软件       | 下载地址                      |
| :--------- | :---------------------------- |
| **WinFsp** | https://winfsp.dev/rel/       |
| **Rclone** | https://rclone.org/downloads/ |

WinFsp 正常安装，Rclone 解压到 `C:\rclone\`，其他位置也行。

#### 2. 配置 Rclone

打开 **CMD** 或 **PowerShell**：

```
C:\rclone\rclone.exe config
```

按提示操作：

```
n        （新建）
cloudreve （名称随便起）
59      （选 WebDAV，不同版本数字可能不同，找 webdav）
http://127.0.0.1:5212/dav   （你的 Cloudreve WebDAV 地址）
8        （选 Other）
你的邮箱   （Cloudreve 用户名）
y        （输入密码）
WebDAV密码 （Cloudreve 生成的 WebDAV 密码）
         （bearer_token 留空，直接回车）
q        （退出）
```

#### 3. 挂载为 Z 盘

```
C:\rclone\rclone.exe mount cloudreve: Z: --vfs-cache-mode full
```

✅ 完成！Z 盘就是你的 Cloudreve

# AList vs Cloudreve 对比

## 🎯 定位不同

|          | AList           | Cloudreve    |
| -------- | --------------- | ------------ |
| **本质** | 文件列表/聚合器 | 完整私有云盘 |
| **存储** | 挂载别人的存储  | 自己管理存储 |
| **定位** | 展示和播放      | 存储和管理   |

---

## 📊 功能对比

| 功能              | AList  | Cloudreve |
| ----------------- | :----: | :-------: |
| 文件上传          | ⚠️ 有限 |  ✅ 完整   |
| 文件管理          | ⚠️ 基础 |  ✅ 完整   |
| 多用户系统        |   ❌    |     ✅     |
| 用户权限          |   ❌    |     ✅     |
| 分享链接          | ⚠️ 简单 |  ✅ 完整   |
| 离线下载          |   ❌    |     ✅     |
| 聚合多网盘        | ✅ 超强 |     ❌     |
| 视频播放          | ✅ 更好 |     ✅     |
| 挂载阿里/百度云盘 |   ✅    |     ❌     |

---

## 🤔 选择建议

### 选 AList 如果：
```
✅ 只是自己一个人用
✅ 想聚合多个网盘统一查看
✅ 主要是看视频/下载
✅ 不需要复杂的文件管理
```

### 选 Cloudreve 如果：
```
✅ 需要多用户/家人共用
✅ 需要完整的文件管理
✅ 需要分享功能
✅ 想搭建真正的私有云盘
```

---

## ✅ 最佳方案：两个都用

```
Cloudreve → 主力存储，管理本地文件
    ↓
  AList → 聚合 Cloudreve + 阿里云盘 + 百度网盘...
    ↓
  手机访问 AList 统一入口
```

这样既有完整的云盘功能，又能聚合所有资源！电脑启动Alist服务器，手机访问电脑5244端口。

# 网络硬盘

WebDAV只适合存储大文件或者用来局域网共享硬盘，不适合作为网络硬盘扩容。

不同挂载方式，效率差异很大：

| 协议    | 性能   | 特点                     |
| ------- | ------ | ------------------------ |
| **SMB** | ⭐⭐⭐⭐   | Windows 原生，稳定       |
| **NFS** | ⭐⭐⭐⭐⭐  | Linux / macOS 性能最好   |
| WebDAV  | ⭐⭐     | 方便但慢                 |
| FTP     | ⭐⭐⭐    | 不适合当“盘”             |
| iSCSI   | ⭐⭐⭐⭐⭐⭐ | **最像真硬盘**，配置复杂 |

👉 如果你想“当扩容盘用”，**WebDAV 是最不合适的**
