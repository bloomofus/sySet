# Unreal Engine 中球面插值 vs 扭曲与摇摆

这两种都是**旋转插值方法**，主要用于骨骼动画和旋转混合，但原理和效果不同。

## 球面插值 (Slerp - Spherical Linear Interpolation)

### 原理
```
在四元数球面上沿最短弧线进行插值
```

### 特点
| 优点         | 缺点                     |
| ------------ | ------------------------ |
| 路径最短     | 可能产生不自然的中间姿态 |
| 角速度恒定   | 不区分旋转分量           |
| 计算相对简单 | 对某些关节效果不理想     |

### 示意图
```
    A ----→ B
     \    /
      \  /      ← 球面上的最短弧线
       \/
```

---

## 扭曲与摇摆 (Swing and Twist)

### 原理
```
将旋转分解为两个独立分量：
- Twist（扭曲）：沿骨骼主轴的旋转（如手臂内外旋）
- Swing（摇摆）：垂直于主轴的旋转（如手臂抬起/放下）
```

### 示意图
```
        Swing（摇摆）
           ↑
           |
    ←------●------→  Swing（摇摆）
           |
           ↓
      
    ↻ Twist（扭曲）= 沿骨骼轴自转
```

### 特点
| 优点               | 缺点           |
| ------------------ | -------------- |
| 更符合人体关节运动 | 计算稍复杂     |
| 可单独控制各分量   | 需要指定扭曲轴 |
| 避免关节"翻转"问题 |                |

---

## 实际对比

### 场景：手臂从 A 姿势到 B 姿势

```
球面插值：
A → → → → B   （直接最短路径，可能穿模或不自然）

扭曲与摇摆：
A → 先调整摇摆 → 再调整扭曲 → B （分步骤，更自然）
```

## 使用建议

| 场景             | 推荐方法   |
| ---------------- | ---------- |
| 简单旋转动画     | 球面插值   |
| 人体手臂/腿部    | 扭曲与摇摆 |
| 肩膀、髋关节     | 扭曲与摇摆 |
| 摄像机旋转       | 球面插值   |
| IK 解算          | 扭曲与摇摆 |
| 需要限制旋转范围 | 扭曲与摇摆 |

## 在 UE 中的设置位置

```
骨骼网格体 → 动画蓝图 → 混合节点 
→ 细节面板 → 旋转模式 (Rotation Mode)
```

选项：
- `Spherical` = 球面插值
- `Swing and Twist` = 扭曲与摇摆

---

**简单总结**：球面插值走"最短路"，扭曲与摇摆更"像人动"。

# 翼片绕错误中心点旋转的问题

> 改好了，但是又有问题，为什么翼片会绕着蓝图中心点转动，而不是绕着自身中心点转动、

物理约束的**约束点位置**在蓝图原点（0,0,0），而不是在翼片与body的连接处。**简单记忆**：物理约束放在哪里，翼片就绕哪里转！
